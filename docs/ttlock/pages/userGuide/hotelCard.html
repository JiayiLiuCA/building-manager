发卡对接

一、发卡方式

1、蓝牙发卡：

调用SDK里面的添加IC卡方法通过蓝牙发卡：APP（SDK）调用添加IC卡方法后，门锁会提示把卡贴到锁上，卡信息将被录入锁里，注意：此需要在锁旁边进行

2、网关发卡：

网关发卡方案是通过读卡器读取IC卡的卡号，将卡号写入锁里，刷卡开锁时，锁识别卡号是否和已经添加到锁里的卡号一致，确定卡能否开锁。

网关发卡方案依赖网关设备，有连接网关的锁，可以调用云端添加IC卡接口，通过网关或WiFi锁将卡号添加到锁里。

3、发卡器发卡：

酒店发卡方案的原理是通过发卡器DLL将锁的MAC地址写入卡里，刷卡开锁时，锁读取卡中的锁MAC与当前锁的MAC比对看是否一致，确定卡能否开锁。

开发者开发电脑发卡软件集成发卡器DLL，通过DLL里的接口，可以实现发卡、退卡、查卡、挂失卡等操作。

网关发卡一次只能把一个卡号写入一把锁里，酒店发卡方式支持发层卡、总卡，并且不依赖云端接口和网关设备，可以在没网络的情况下发房间卡，比网关发卡更方便快捷，但需要集成发卡器DLL，开发复杂度更高，开发者可根据产品需求选择合适的方案。

注意：使用demo测试的时候，请使用此路径下的程序：cardencoderdemo-master\Release\CardEncoderDemo.exe

发卡器发卡对接两套方案选择：

方案一：仅开发电脑发卡软件。使用通用的“通通酒店”APP对锁进行添加和管理。

方案二：不依赖通通酒店App，根据SDK开发自己的APP对锁进行添加和管理， 也对接发卡器发卡。

方案一：依赖通通酒店进行发卡对接

方案一是最简单快速的对接方式，开发者只需关注发卡功能，而对锁的添加、管理则使用通用的“通通酒店”APP来实现。步骤如下：

1、在通通酒店APP里添加锁

下载“通通酒店”APP（https://hotel.sciener.com/），并在APP里申请开通锁管理员账号。这个账号用来登录APP，并作为这个酒店所有锁的管理员。在APP里，除了添加锁外，还可以设置锁的楼栋编号、楼层编号，可以设置要使用的卡片扇区信息，可以校准锁的时间，也可以收集锁的操作记录。所有需要跟锁直接通讯的功能，都可以通过这个APP完成。

2、获取client_id和client_secret

登录通通酒店管理员账号，在通通酒店客户端设置设置选项里点击门锁对接，点击申请对接后获取client_id和client_secret

3、获取访问令牌

用门锁对接界面显示的client_id和client_secret，登录通通酒店使用的账号和密码调用云端接口获取访问令牌（accessToken)，注意accessToken会在expires_in标识的时间后失效，失效后需要重新获取或刷新

注意：通通酒店账号对应的clientId不能访问科技侠账号，只能通过步骤3中通通酒店账号获取访问令牌。

4、集成发卡器DLL发卡

发卡器DLL可集成到发卡软件里。参考发卡器DLL使用手册，通过DLL里的接口，可以实现发卡、退卡、查卡、挂失卡等操作。

对于锁的基本信息的查询，以及对密码及电子钥匙的获取，可以通过云端的接口（Cloud API V3，携带步骤4中获取的访问令牌)来实现。其中，锁的楼栋楼层号信息无法通过开放平台接口获取，需要在门锁对接界面导出锁数据获取。

方案二：不依赖通通酒店进行发卡对接

这是一个完整的对接方案，既对接APP SDK（V3)， 也对接发卡器发卡。具体步骤如下：

1、注册开发者账号

2、创建应用获取client_id和client_secret

应用创建后默认是审核中状态，后台会在2个工作日内审核并邮件通知开发者。

通过后开发者可以在应用详情页查看client_id和client_secret，请妥善保管自己的client_id和client_secret.

如果有多个项目，一个开发者账号可以创建多个应用。

一个开发者账号可以创建多个应用，一个酒店对应一个应用。

3、基于APP SDK开发APP添加锁

类似于方案一中的通用APP，开发者自己开发APP来管理锁，需要在添加锁时将酒店信息（hotelInfo)以及楼栋楼层号信息写入锁里。

4、获取访问令牌

用通通锁或者通过接口创建的账号和密码调用云端接口获取访问令牌（accessToken)，注意accessToken会在expires_in标识的时间后失效，失效后需要重新获取或刷新

5、集成发卡器DLL发卡

发卡器DLL可集成到发卡软件里。参考上述所说发卡器DLL使用手册，通过DLL里的接口，可以实现发卡、退卡、查卡、挂失卡等操作。

 

对于锁的基本信息的查询，以及对密码及电子钥匙的获取，可以通过云端的接口（Cloud API V3，携带步骤4中获取的访问令牌)来实现。其中，锁的楼栋楼层号信息通过开放平台接口无法获取，需要开发者自行进行记录保存。

四、常见问题

  1. 如何保证酒店间的卡不能串用？

    每个酒店有单独client_id对应一个应用，每个应用对应一个加密方式。

  2. 方案一如何修改酒店管理员账号？

    登录通通酒店APP或客户端，在APP或客户端设置中修改账号。

  3. 方案一锁的楼栋编号、楼层编号如何获取？

    目前开放平台无相关接口，可在通通酒店客户端设置---门锁对接界面导出。

  4. 方案一锁的MAC地址如何获取？

    用酒店管理员账号调开放平台获取锁相关信息的接口即可获取，也可以通过门锁对接界面导出。
