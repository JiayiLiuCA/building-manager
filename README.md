# 和美物业 AI OS — 产业园演示(第二版)

纯前端演示项目:**不接任何后端 / API**,集中 mock 数据层 + 内存 store 模拟真实增删改。
物业端(主管/客服)与企业端共享同一份内存数据,跨端操作实时联动;**刷新页面即重置**为初始演示数据(不做持久化)。

业务场景:**和美产业园**(A/B/C 三区 · 14 栋 · 30 家入驻企业,其中 10 家整栋独占、20 家分布于多户楼栋)。
固定「今天」锚点为 **2026-06-06**:超时时长、欠费月数、趋势与同环比等所有相对数据在任何日期演示都一致。

## 快速开始

```bash
npm install
npm run dev        # http://localhost:5173
```

## 推荐演示路线(六条核心链路)

1. **报修闭环 + 满意度联动**:企业①发起报修 → 物业端 维修工单 接单→派单→预约→录入完工 → 企业端电子签字关单 → 评价 → 驾驶舱/满意度页实时联动。
2. **发布停电通知 → 企业首页可见**:物业端 通知管理 新建停电通知(客服可选范围仅限管辖企业/区/楼栋)→ 切到范围内企业账号,首页顶部醒目展示;预置的「B 区停电检修通知」登录 `company2` 即见。公共区域维修工单详情可**一键生成关联通知**。
3. **上传发票 → 企业端查询下载**:企业档案 → 发票管理 上传(mock 仅记文件名)→ 切对应企业 → 发票查询 筛选并下载 PDF。
4. **主管调整权限 → 客服驾驶舱实时变化**:`admin` 进 权限设置,把企业③(洄澜餐饮管理)勾给王琳并保存 → 切 `cs_wang`(名下 21→22 家)与 `cs_liu`(9→8 家),驾驶舱/日报/各列表全部即时按新范围聚合。
5. **收款跟进三色实时联动(核心卖点)**:驾驶舱/企业档案 → 收款跟进:企业② 因「超时报修 + 未闭环投诉」判 🟠 暂缓;物业完结工单并闭环投诉 → 实时变 🟢 建议跟进。三依据(缴费习惯/报修关单/投诉关单)+ 历史减免逐企业并列展示;⚪️ 暂不跟进 = 未到习惯付款日或无习惯记录待沟通。
6. **任务穿透 + 核抄同环比**:工作任务清单 年→季→月→周 四级穿透下钻,勾选「标记完成」达成率实时变化(客服仅见自己的任务);能耗核抄 购水/购电 24 个月数据,月度用量趋势 + 同比/环比显性展示。

## 技术栈

Vite + React 19 + TypeScript · Tailwind CSS v4 + shadcn/ui(Radix)· Zustand(内存 store)· react-router v7 · Recharts(全部图表常显数值 + 明细小表)· lucide-react

## 目录结构

```
src/
  lib/            日期锚点(2026-06-06)/期间口径(年/季/月)/图表数值工具/状态映射(statusMaps)
  data/
    types.ts      全部实体契约(工单/投诉状态由事件数组派生,单一事实源)
    store.ts      单一 Zustand store:三角色全部动作(登录/工单/投诉/缴费/通知/发票/调研/任务/权限/跟进…)
    seed/         固定种子数据:3 区 14 栋 30 家企业、12 个月四费类账单、目标、维保/巡检/核抄(24 月)/任务、故事企业编排(storyCompanies.ts)
    selectors/    派生逻辑:scope.ts 权限唯一入口、收缴/达成聚合、收款跟进三色、满意度双来源、日报实时生成
    mock-content/ AI 摘要与日报文案模板(函数化,吃派生数字)、AI 客服规则(园区语境)
  layouts/        角色守卫 + 物业端(分组侧栏)/ 企业端(单列)两套布局
  components/     ui/(shadcn) shared/(KpiCard/ChartCard 带明细表插槽/StatusStepper 双步序/BillingHistory 四费类…)
  pages/          property/(驾驶舱/日报/经营管理×4/服务品质×3/内控×3/通知/企业档案/权限) company/(首页/报修/账单/发票/调研/AI/信息)
docs/             task-brief.md(任务书)client-requirements.md(客户原始需求)
```

## 自测脚本

```bash
npm run verify:seed    # seed 不变量硬断言(≈95 条:三视角一致性、跨模块口径、权限隔离、三色摆位…)
npm run verify:flows   # 直接驱动 store 走完 10 条跨端联动链路(报修闭环/三色联动/权限改配/通知/发票/调研…)
npm run smoke          # Playwright 真浏览器遍历两端全部页面(需 dev server 运行中)+ 截图到 scripts/shots/
```
