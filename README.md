# 和美物业 AI OS — 前端演示 MVP

纯前端演示项目:**不接任何后端 / API**,集中 mock 数据层 + 内存 store 模拟真实增删改。
物业端与业主端共享同一份内存数据,跨端操作实时联动;**刷新页面即重置**为初始演示数据(不做持久化)。

## 快速开始

```bash
npm install
npm run dev        # http://localhost:5173
```

## 推荐演示路线(六条核心链路)

1. **报修闭环**:业主(张伟)发起报修 → 物业端 工单/投诉 接单→派单→预约→录入完工 → 业主电子签字关单 → 评价 → 驾驶舱满意度联动。
2. **催缴前置判断(核心卖点)**:催缴页三色建议 —— 🟢张伟(服务到位仍欠费=建议催缴)/ 🟠李强(超时漏水工单+未闭环投诉=暂缓催缴)/ ⚪️王秀兰(12个月无水电用量=疑似空置待核实)。
3. **暂缓→建议 实时联动**:物业完结李强的超时工单 + 回复并关闭其投诉 → 催缴页李强从 🟠 实时变 🟢。
4. **催缴→弹窗→缴费**:物业对张伟「发起催缴」→ 切到业主端张伟 → 登录即弹催缴通知 → 立即缴费 → 物业端收缴率/欠费清单实时变化,催缴记录自动解决。
5. **空置核实**:王秀兰户档案 → 核实并登记空置 → 未缴物业费按半价校准、生成「停水停电」待办、⚪️ 消失。
6. **投诉升级链**:业主投诉(可关联工单)→ 物业派至责任部门 → 回复 → 业主不满意申请主管介入 → 主管回复 → 关闭(周杰户长期欠费已上报,见驾驶舱风险清单与日报)。

固定「今天」锚点为 **2026-06-06**:超时时长、欠费月数、趋势图等所有相对数据在任何日期演示都一致。

## 技术栈

Vite + React 19 + TypeScript · Tailwind CSS v4 + shadcn/ui(Radix)· Zustand(内存 store)· react-router v7 · Recharts · lucide-react

## 目录结构

```
src/
  lib/            日期/金额格式化、固定种子随机、ID 生成、状态枚举→中文与配色(statusMaps)
  data/
    types.ts      全部实体接口(工单/投诉状态由事件数组派生,单一事实源)
    store.ts      单一 Zustand store:两端全部动作(登录/工单/投诉/缴费/空置/催缴/客服…)
    seed/         固定种子数据:3 小区 112 户、12 个月账单、故事户编排(storyHouseholds.ts)
    selectors/    派生逻辑:状态机、超时判定、收缴聚合、催缴三色规则、驾驶舱指标
    mock-content/ AI 摘要、7 天日报与钉钉原始条目、AI 客服规则(演示前可直接改文案)
  layouts/        角色守卫 + 物业端(侧栏)/ 业主端(单列)两套布局
  components/     ui/(shadcn 生成) shared/(KpiCard/Timeline/StatusStepper/三色徽章…)
  pages/          property/(驾驶舱/日报/工单投诉/催缴/缴费钻取/户档案) resident/(缴费/报修/投诉/AI咨询/我的)
```

## 自测脚本

```bash
npm run verify:seed    # seed 数据不变量(收缴率≈87.6%、故事户三色建议、超时工单数…)
npm run verify:flows   # 直接驱动 store 走完六条跨端联动链路
npm run smoke          # Playwright 真浏览器遍历两端页面(需 dev server 运行中)+ 截图到 scripts/shots/
```
